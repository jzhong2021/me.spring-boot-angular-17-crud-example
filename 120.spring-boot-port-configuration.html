Spring Boot Port Configuration

Yes, 8080 is the default port for Spring Boot.

When you don't specify a port in your Spring Boot application configuration, it will automatically use port 8080.

How to Change the Port

Method 1: Using application.properties

Add the following line to your application.properties file:

server.port=8081

File location: spring-boot-server/src/main/resources/application.properties

Method 2: Using application.yml

If you're using YAML configuration instead:

server:
  port: 8081

Method 3: Command Line Argument

You can also specify the port when running the application:

java -jar your-app.jar --server.port=8081

Or with Maven:

mvn spring-boot:run -Dspring-boot.run.arguments=--server.port=8081

Method 4: Environment Variable

Set the port using an environment variable:

export SERVER_PORT=8081

Or in Windows PowerShell:

$env:SERVER_PORT=8081

Complete Configuration Steps for This Project

If you want to change from port 8080 to 8081, you need to update two places:

1. Spring Boot Configuration

File: spring-boot-server/src/main/resources/application.properties

Add:

server.port=8081

2. Angular Service Configuration

File: angular-17-client/src/app/services/tutorial.service.ts

Change line 6 from:

const baseUrl = 'http://localhost:8080/api/tutorials';

To:

const baseUrl = 'http://localhost:8081/api/tutorials';

3. CORS Configuration (if needed)

File: spring-boot-server/src/main/java/.../controller/TutorialController.java

Ensure CORS allows your Angular frontend:

@CrossOrigin(origins = "http://localhost:4200")

Note: CORS should allow the Angular frontend port (4200), not the backend port.

Default Ports Summary

Application Default Port Configuration File
Spring Boot 8080 application.properties
Angular (ng serve) 4200 package.json or command line
Tomcat (embedded) 8080 application.properties

After Making Changes

  1. Stop the Spring Boot server (Ctrl+C)
  2. Restart Spring Boot: cd spring-boot-server && .\mvnw.cmd spring-boot:run
  3. Restart Angular (if you changed the service URL): cd angular-17-client && npm start
  4. Test the API: http://localhost:8081/api/tutorials
Important: If you change the Spring Boot port, make sure to update the Angular service URL to match, otherwise the frontend won't be able to communicate with the backend.

Verification

To verify which port Spring Boot is running on, check the console output when starting the application. You should see something like:

Tomcat started on port(s): 8081 (http) with context path ''

Or if using the default:

Tomcat started on port(s): 8080 (http) with context path ''
